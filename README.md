# 🥋 Ouro Negro - Sistema de Cadastro Capoeira Nagô

O projeto **Ouro Negro** é uma aplicação web desenvolvida para gerenciar as inscrições e a divulgação da oficina de movimentação da **Capoeira Nagô em Monte Carmelo, MG**. O sistema combina uma interface moderna e responsiva com um backend robusto para garantir uma experiência premium aos usuários.

## 🚀 Tecnologias Utilizadas

Este projeto foi construído utilizando as seguintes tecnologias:

* **Frontend:** [React](https://reactjs.org/) com [Vite](https://vitejs.dev/) para um desenvolvimento rápido e otimizado.
* **Animações:** [Framer Motion](https://www.framer.com/motion/) para transições suaves e efeitos de scroll reveal.
* **Estilização:** CSS3 puro com foco em design responsivo e estética "Preto & Ouro".
* **Backend:** [Node.js](https://nodejs.org/) para processamento de dados e integração de API.
* **Infraestrutura:** [Docker](https://www.docker.com/) para padronização do ambiente de desenvolvimento e deploy.

## ✨ Funcionalidades Principais

* **Carrossel de Ministrantes:** Exibição dinâmica de mestres e professores com transições automáticas a cada 3 segundos.
* **Footer Institucional:** Layout responsivo com informações da produtora cultural **Akaésse Kaonny**, organização e coordenação (**Contramestre Negosann**).
* **Formulário de Inscrição:** Sistema de validação de dados para participantes da oficina.
* **Design Mobile-First:** Interface totalmente adaptada para dispositivos móveis, mantendo a hierarquia visual.

## 📦 Como Executar o Projeto

### Pré-requisitos
* Node.js instalado
* Docker (opcional, para ambiente isolado)

### Passo a Passo
1.  **Clone o repositório:**
    ```bash
    git clone [https://github.com/YuryTuratti/ouro_negro.git](https://github.com/YuryTuratti/ouro_negro.git)
    ```
2.  **Instale as dependências:**
    ```bash
    npm install
    ```
3.  **Inicie o servidor de desenvolvimento:**
    ```bash
    npm run dev
    ```

## 📂 Estrutura de Pastas
A estrutura segue o padrão escalável para projetos React:
* `src/components/`: Componentes reutilizáveis da interface.
* `src/styles/`: Arquivos de estilização global.
* `public/`: Ativos estáticos como imagens e fotos dos ministrantes.
